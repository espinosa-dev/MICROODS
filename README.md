# MICROODS

> **Jugando por un futuro sostenible.** > Una plataforma web interactiva compuesta por microjuegos diseñados para concienciar y educar sobre los **Objetivos de Desarrollo Sostenible (ODS)** de las Naciones Unidas.

---

## Sobre el Proyecto

**MICROODS** es un proyecto educativo e interactivo que busca acercar los Objetivos de Desarrollo Sostenible a los usuarios. Cada microjuego está enfocado en un ODS específico (como Acción por el Clima, Educación de Calidad, o Igualdad de Género), ofreciendo una experiencia divertida que fomenta el aprendizaje y la acción social.

## Características Principales

* **Microjuegos Temáticos:** Diversos minijuegos, cada uno centrado en enseñar los valores y metas de un ODS particular.
* **Interfaz Intuitiva:** Diseño atractivo y fácil de usar, pensado para usuarios de todas las edades.
* **Educativo y Divertido:** Combina mecánicas de juego sencillas con información valiosa sobre sostenibilidad.
* **Totalmente Responsivo:** Adaptable a dispositivos móviles, tablets y escritorio.

## Tecnologías y Herramientas

Este proyecto ha sido desarrollado utilizando tecnologías web estándar y buenas prácticas de trabajo colaborativo:

### Front-end
* **HTML5:** Estructuración semántica del contenido.
* **CSS3:** Estilos, animaciones y diseño responsivo (Grid/Flexbox).
* **JavaScript:** Lógica de los microjuegos y manejo de eventos.

### Gestión y Control de Versiones
* **Git:** Control de versiones local.
* **GitHub:** Alojamiento del repositorio y trabajo colaborativo.
* **Kanban (GitHub Projects):** Planificación ágil, asignación de tareas y seguimiento del progreso del proyecto (Backlog, In Progress, Review, Done).

## Metodología de Trabajo
Para asegurar un desarrollo continuo y organizado, he usado la metodología de **Kanban**. 

Esto nos permite:

- Dividir el desarrollo de cada microjuego en tareas pequeñas y manejables.
- Mantener un control visual del estado de cada funcionalidad.
- Asegurar que las fases de diseño, lógica JS y pruebas se cumplan antes de integrar cambios a la rama principal (main).

## Contribuciones
Si tienes una idea para un nuevo microjuego de un **ODS** o quieres mejorar el código existente:

1. Haz un Fork del proyecto.
2. Crea tu rama de características (git checkout -b feature/NuevoMicrojuego).
3. Haz Commit de tus cambios (git commit -m 'Añade nuevo microjuego sobre ODS 14').
4. Haz Push a la rama (git push origin feature/NuevoMicrojuego).
5. Abre un Pull Request.
